# Diet Mate — Бриф проекта

## Контекст

Пользователь проходит курс по похудению Physical Transformation с куратором. Куратор еженедельно настраивает рацион, контролирует ключевые параметры питания. Курс заканчивается — нужен инструмент, который заменит внешний контроль и не даст скатиться обратно в хаос.

## Цель продукта

Веб-приложение, которое:
- Подтягивает данные о питании из Fatsecret
- Показывает прогресс по ключевым параметрам рациона
- Позволяет ежедневно вводить вес и шаги
- Формирует еженедельные отчёты

## Параметры контроля

| Параметр | Единица | Источник данных |
|---|---|---|
| Калории | ккал/день | Fatsecret API (напрямую) |
| Растительность (овощи + фрукты + зелень) | г/день | Fatsecret + LLM категоризация |
| Авокадо | г/день | Fatsecret + совпадение по названию |
| Кальций | мг/день | Fatsecret API (напрямую) |
| Омега-3 | г/день | USDA FoodData API по названию продукта |
| Яйца | шт/день | Fatsecret + LLM категоризация (50г = 1 шт) |
| Морепродукты | порций/день | Fatsecret + LLM категоризация (100-120г = 1 порция) |
| Вес | кг | Ручной ввод |
| Шаги | шт | Ручной ввод |

Все целевые значения настраиваются пользователем в интерфейсе (не хардкод).

## Экраны приложения

### 1. Дашборд (главный экран)
- Поля ручного ввода: вес (кг), шаги
- 7 прогресс-баров с текущим / целевым значением
- Данные за сегодня, обновляются из Fatsecret

### 2. Настройки целей (Settings)
- Редактируемые целевые значения по каждому параметру
- Сохраняются в БД

### 3. Еженедельный отчёт
- График веса по дням
- Средние шаги за неделю
- По каждому параметру питания: сколько дней из 7 попал в цель

## Технический стек

| Слой | Технология |
|---|---|
| Фронт + API | Next.js |
| Авторизация питания | Fatsecret OAuth 2.0 |
| Категоризация продуктов | Claude API (LLM) + кэш в БД |
| Омега-3 | USDA FoodData API (бесплатный) |
| База данных | SQLite или Supabase |

### Логика категоризации продуктов
1. Продукт из Fatsecret поступает в систему впервые → Claude API классифицирует категорию
2. Результат сохраняется в локальный кэш
3. При повторном появлении продукта — берём из кэша без API-запроса
4. Пользователь может скорректировать категорию вручную

### Категории LLM
- Растительность (овощи / фрукты / зелень)
- Авокадо
- Яйца
- Морепродукты / рыба
- Другое

## Формат POC

Первая версия — веб-приложение. Шаги вводятся вручную.

Гаджет пользователя: Garmin Instinct 2 → Garmin Connect → Apple Health (HealthKit).
Автоматическое подтягивание шагов — в iOS версии (следующий этап после валидации POC).
